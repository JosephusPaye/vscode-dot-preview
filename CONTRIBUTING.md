# Contributing

## Setup

- install the recommended extensions (`amodio.tsl-problem-matcher`, `ms-vscode.extension-test-runner`, and `dbaeumer.vscode-eslint`)

## Get up and running the Web Extension

- Run `yarn install`.
- Optionally place breakpoints in the code anywhere in the `src/web/` folder.
- Build and run with debugging enabled by pressing F5 (Run Web Extension).
- This will open a new VS Code window with the extension loaded
- Test the extension by opening some DOT files (there are sample files in the `samples/` folder).

## Make changes

- Relaunch the extension from the debug toolbar after changing code
- You can also reload the VS Code window with the extension to load changes
